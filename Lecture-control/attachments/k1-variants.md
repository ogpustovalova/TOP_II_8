# Рубежный срез К1: варианты

Каждый вариант содержит четыре коротких вопроса, две расчётные задачи и одну
ситуационную задачу. Все промежуточные вычисления должны быть показаны.

## Вариант 1

### Часть А. Термины и концепции

1. Почему композиция нескольких линейных слоёв без нелинейной активации
   эквивалентна одному линейному преобразованию?
2. Почему в PyTorch перед `CrossEntropyLoss` обычно не применяют `Softmax`?
3. Зачем вызывать `optimizer.zero_grad()` перед обратным проходом?
4. Какое практическое ограничение имеет универсальная теорема аппроксимации?

### Часть Б. Расчётные задачи

1. Рассчитайте число обучаемых параметров MLP `784 → 64 → 10`. У обоих слоёв
   есть смещения.
2. Для модели `ŷ = wx + b` даны `x=2`, `y=0`, `w=0,5`, `b=0` и
   `L=0,5(ŷ-y)²`. Выполните один шаг градиентного спуска с `lr=0,1`: найдите
   `L`, градиенты по `w` и `b`, новые значения параметров.

### Часть В. Ситуационная задача

MLP достигает 99 % accuracy на train, 68 % на validation; train loss продолжает
снижаться, а validation loss после 12-й эпохи растёт. Поставьте диагноз,
предложите порядок проверок и не менее трёх изменений протокола. Объясните,
какие графики и метрики подтвердят улучшение.

## Вариант 2

### Часть А. Термины и концепции

1. Чем насыщение sigmoid отличается от поведения ReLU при положительных входах?
2. Почему He обычно используют с ReLU, а Xavier — с tanh?
3. В чём компромисс mini-batch SGD между полным batch и обновлением по одному
   объекту?
4. Почему параметры нормализации нельзя вычислять по train и test вместе?

### Часть Б. Расчётные задачи

1. Рассчитайте число обучаемых параметров MLP `8 → 64 → 32 → 1`, включая
   смещения.
2. Для логитов двух классов `z=[ln 2, 0]` вычислите softmax-вероятности и
   cross-entropy, если истинный класс — второй.

### Часть В. Ситуационная задача

Исследователь сначала нормализовал весь датасет, затем случайно разделил его на
train, validation и test. Полученная test-ошибка заметно ниже результата после
развёртывания. Объясните источник и направление смещения оценки. Предложите
корректный конвейер подготовки данных и проверки модели. Назовите ещё две
возможные причины разрыва между test и эксплуатацией и объясните, как отличить
их экспериментально.

## Вариант 3

### Часть А. Термины и концепции

1. Зачем MLP нужна нелинейность между полносвязными слоями?
2. Чем должен отличаться выходной слой для регрессии и многоклассовой
   классификации?
3. Почему Dropout ведёт себя по-разному в режимах обучения и инференса?
4. Как слишком большая и слишком малая скорость обучения проявляются на кривой
   loss?

### Часть Б. Расчётные задачи

1. Рассчитайте число обучаемых параметров MLP `784 → 128 → 64 → 10`, включая
   смещения.
2. Для бинарного нейрона `z=wx+b`, `p=sigmoid(z)` даны `x=2`, `y=1`, `w=0`,
   `b=0`. Для binary cross-entropy используйте
   `∂L/∂z = p-y`. Выполните один шаг с `lr=0,1` и найдите новые `w` и `b`.

### Часть В. Ситуационная задача

При обучении MLP loss через несколько шагов становится `NaN`. Сформулируйте не
менее четырёх проверяемых гипотез, расположите проверки в разумном порядке и
предложите исправления. Укажите, какие диагностические значения нужно
логировать.
