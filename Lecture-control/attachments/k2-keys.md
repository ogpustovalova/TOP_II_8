# Рубежный срез К2: ключи для преподавателя

> Не включать этот файл в студенческий комплект контрольной.

Ответ может быть сформулирован иначе. Уровень по каждому критерию определяется
по полноте перечисленных элементов и рубрике.

## Вариант 1

### Часть А

1. Остаточный путь добавляет вход блока к преобразованию и создаёт короткий путь
   для сигнала и градиента, упрощая обучение глубоких сетей.
2. Max pooling выбирает максимальный локальный отклик, уменьшает разрешение и
   повышает устойчивость к малым сдвигам, но теряет точное положение и слабые
   признаки.
3. LSTM хранит hidden state и отдельное cell state с входным, забывающим и
   выходным гейтами; GRU объединяет память в hidden state и использует меньше
   гейтов.
4. KL приближает распределение латентных кодов к выбранному prior, делает
   пространство более регулярным и пригодным для генерации.

### Часть Б

1. После свёртки: **`32×32×32`**; после pooling: **`16×16×32`**.
   Параметры: `32×(3×3×3+1)=` **896**.
2. При одном bias на гейт: `4h(d+h+1)=4×32×(64+32+1)=` **12 416**.
   Формула PyTorch с двумя bias на гейт отличается; студент получает полный
   балл, если явно указал и последовательно применил библиотечную конвенцию.

### Часть В

Нужны фиксированные стратифицированные train/validation/test, выбор 20 объектов
только из train, одинаковая голова и преобразования, ImageNet-нормализация.
Сначала обучается голова frozen backbone, затем отдельная копия или сохранённое
состояние fine-tuning с малым lr. Выбор делается по validation, test используется
после выбора. Сравниваются accuracy, время и число обучаемых параметров. Риски:
переобучение, дисбаланс, утечка test, слишком большой lr и нестабильность seed.

## Вариант 2

### Часть А

1. Свёртка разделяет веса между позициями и использует локальную связность;
   полносвязный слой имеет отдельный вес для каждой пары вход–выход.
2. Заморозка сохраняет предобученные признаки и уменьшает число настраиваемых
   параметров на малой выборке. Малый lr при fine-tuning снижает риск разрушить
   полезные веса backbone и катастрофического забывания.
3. Токенизация выделяет единицы текста, морфология определяет формы и леммы,
   синтаксический анализ строит связи `head/dependency` между токенами.
4. AdamW отделяет weight decay от адаптивного шага по градиенту; обычное
   добавление L2 к loss проходит через адаптивное масштабирование Adam.

### Часть Б

1. `floor((64-5+2×2)/2)+1=32`; выход **`32×32×16`**. Параметры:
   `16×(3×5×5+1)=` **1 216**.
2. `3h(d+h+1)=3×64×(50+64+1)=` **22 080** параметров при одном bias на
   гейт. Библиотечная конвенция с двумя bias принимается при явном пояснении.

### Часть В

Есть переобучение и утечка словаря. Сначала выполняется стратифицированное
разбиение, затем словарь и все обучаемые преобразования строятся только по train;
validation выбирает архитектуру/hidden size, test используется один раз.
Regex, токенизация, PAD/OOV и длина должны совпадать для всех частей. LSTM, GRU и
1D-CNN обучаются с одинаковыми seed, бюджетом эпох и early stopping; сравниваются
validation/test accuracy, время и параметры. Уместны Dropout, уменьшение модели
и анализ ошибок.

## Вариант 3

### Часть А

1. Regex заменяет или выделяет шаблоны и возвращает нормализованный текст;
   морфологический анализ определяет формы/леммы токенов; синтаксический анализ
   возвращает дерево или таблицу связей `token — head — dependency`.
2. Репараметризация выносит случайность в `ε`, сохраняя дифференцируемость по
   параметрам `μ` и `σ`.
3. Scheduler меняет lr по времени или метрике, ускоряя начальное обучение и
   позволяя точнее донастраивать параметры позже.
4. Проверка save/load обнаруживает сохранение не того состояния, несовместимую
   конфигурацию или отличие предобработки и подтверждает воспроизводимость.

### Часть Б

1. `floor((28-3+2×1)/2)+1=14`; выход **`14×14×32`**. Параметры:
   `32×(1×3×3+1)=` **320**.
2. Общий слой: `10×8+8=88`; каждая голова: `8×2+2=18`; всего
   **124 параметра**. Для первой координаты KL-вклад 0; для второй выражение в
   скобках равно `-1`. `D_KL=-0,5×(-1)=` **0,5**.

### Часть В

Случайное перемешивание соседних окон перед разбиением создаёт временную утечку:
почти одинаковые участки ряда попадают в train и test. Требуется хронологическое
разбиение train→validation→test и статистики нормализации только по train. Окна
формируются после задания границ так, чтобы вся пара `(input, target)` оставалась
в одной части; при пересекающихся горизонтах вводится gap. Гиперпараметры
выбираются по validation. Наивный прогноз последним доступным или сезонным
значением считается в том же горизонте; test сравнивается один раз после выбора
модели. LSTM и GRU с `hidden_size=32/64` обучаются на одинаковых окнах, seed,
бюджете эпох и критерии early stopping; конфигурация выбирается по validation,
после чего выбранные модели однократно сравниваются на test по MAE и времени.
