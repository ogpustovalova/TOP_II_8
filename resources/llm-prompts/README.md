## Примеры промптов для использования в курсе

| Назначение промпта | Промпт (пример) | Входные данные | Ожидаемый формат результата | Ограничения | Способ верификации |
|---|---|---|---|---|---|
| **Объяснение концепции нейронной сети** | *«Объясни, как работает свёрточная нейронная сеть, используя аналогию с обработкой изображений человеком. Опиши слои свёртки, пулинга и полносвязный слой. Приведи пример архитектуры (например, ResNet) и объясни, зачем нужны skip-connections. Для подтверждения понимания задай 3 контрольных вопроса по теме.»* | Название концепции (например, «свёрточная нейронная сеть») | Структурированное объяснение с аналогиями, описанием компонентов, примером архитектуры и контрольными вопросами | Модель может дать упрощённое или неточное объяснение для сложных концепций[reference:7] | Сравнение с материалами лекций и учебниками (Goodfellow, Bishop); проверка контрольных вопросов |
| **Генерация кода для нейросети** | *«Напиши код на PyTorch для многослойного перцептрона (MLP) с двумя скрытыми слоями для классификации изображений MNIST. Включи: загрузку данных, определение модели, функцию потерь, оптимизатор (Adam), цикл обучения на 10 эпох и визуализацию кривых обучения. Добавь комментарии к каждой части.»* | Описание задачи (MLP для MNIST) | Полный Python-код с комментариями, готовый к запуску в Jupyter Notebook | Код может содержать ошибки или устаревшие конструкции[reference:8] | Запуск кода, проверка работоспособности, соответствия заданию и достижения целевой метрики (accuracy ≥ 0.95) |
| **Отладка и оптимизация модели** | *«Моя модель на PyTorch переобучается на тренировочных данных: accuracy на валидации ≈ 0.85, на тренировке ≈ 0.99. Предложи 3 способа борьбы с переобучением (кроме Dropout) и объясни, как их применить. Напиши фрагмент кода для реализации каждого способа.»* | Описание проблемы (переобучение) | Список методов с пояснениями и примерами кода | Модель может предложить общие или неэффективные методы | Проверка предложенных методов на практике, сравнение метрик до и после применения |
| **Анализ ошибок модели** | *«У меня модель классификации изображений (CNN) показывает низкую точность на классе «кошка» (0.65) и высокую на классе «собака» (0.92). Проанализируй возможные причины: дисбаланс классов, особенности данных, архитектура, гиперпараметры. Предложи план действий для улучшения.»* | Описание проблемы (низкая точность на классе) | Анализ причин и пошаговый план улучшения с обоснованием | Анализ может быть поверхностным | Сравнение с рекомендациями из литературы (например, CS231n); проверка на практике |
| **Проектирование архитектуры** | *«Спроектируй архитектуру нейросети для задачи сегментации медицинских изображений (размер 256×256, 3 класса). Опиши выбор архитектуры (U-Net, DeepLab, или другая), объясни, почему она подходит, укажи функцию потерь, метрики оценки и ожидаемый размер модели. Приведи пример кода на PyTorch.»* | Описание задачи (сегментация медицинских изображений) | Обоснованный выбор архитектуры, описание компонентов, метрик и пример кода | Модель может предложить неподходящую архитектуру[reference:9] | Сравнение с известными решениями (U-Net, DeepLab); проверка на практике (если реализовано) |
| **Генерация тестовых вопросов** | *«Сгенерируй 10 тестовых вопросов (5 закрытых, 5 открытых) по теме "Механизм внимания и трансформеры". Вопросы должны проверять понимание: (1) математической основы, (2) архитектуры, (3) применения в NLP. Для закрытых вопросов укажи правильный ответ и пояснение.»* | Тема (механизм внимания и трансформеры) | 10 вопросов с ответами и пояснениями | Вопросы могут быть слишком простыми или сложными | Проверка преподавателем на соответствие уровню сложности и темам курса |
| **Объяснение результатов эксперимента** | *«Я обучил ResNet-18 на CIFAR-10 и получил accuracy 0.93. Объясни, какие факторы могли повлиять на результат: аугментация данных, регуляризация, оптимизатор, количество эпох. Что можно изменить, чтобы достичь accuracy ≥ 0.95? Предложи конкретные шаги и обоснуй их.»* | Описание эксперимента и результатов | Анализ факторов и план улучшения с обоснованием | Анализ может не учитывать все нюансы | Проверка на практике, сравнение с SOTA-результатами для CIFAR-10 |

## Требования к добавлению

Для каждого сервиса укажите модель/сервис, назначение промпта, входные данные, ожидаемый формат результата, ограничения и способ верификации ответа.

**Пример описания для ChatGPT:**

- **Модель/сервис:** ChatGPT (OpenAI GPT-4/GPT-4o)
- **Назначение:** Объяснение концепций, генерация кода, отладка, помощь в проектировании архитектур.
- **Входные данные:** Текстовый запрос с описанием проблемы или задачи.
- **Ожидаемый формат результата:** Структурированный ответ с пояснениями, кодом (при необходимости) и рекомендациями.
- **Ограничения:** Бесплатная версия имеет ограничение на количество запросов; может генерировать ошибочный код или неточные объяснения.
- **Способ верификации:** Сравнение с эталонными источниками (учебники, документация), проверка кода на выполнимость, экспертная оценка преподавателем.
