## Требования к добавлению

Для каждого бенчмарка укажите:

- **Задача** – что именно оценивается (классификация, генерация, производительность обучения и т.д.).
- **Метрики** – какие числовые показатели используются (accuracy, FID, время обучения и др.).
- **Протокол эксперимента** – фиксированное разбиение данных, аугментации, количество запусков, предобработка.
- **Базовые решения (бейзлайны)** – эталонные модели с известными результатами, с которыми сравниваются новые подходы.
- **Требования к воспроизводимости** – указание случайных сидов, версий библиотек, конфигураций обучения.
- **Правила сравнения** – как интерпретировать результаты (например, статистическая значимость, доверительные интервалы).

**Пример описания для CIFAR-10:**

- **Задача:** Классификация изображений по 10 классам.
- **Метрики:** Top-1 accuracy (%) на тестовой выборке.
- **Протокол эксперимента:** Разбиение 50 000 / 10 000. Аугментации: случайные горизонтальные отражения, сдвиг до 4 пикселей, нормализация по среднему и std каналов. Обучение: SGD с моментом 0.9, распад learning rate по косинусу, 200 эпох, размер батча 128.
- **Базовые решения (бейзлайны):**
  - Простой CNN (2 свёрточных слоя + 2 полносвязных) — ~70%.
  - ResNet-18 — ~95%.
  - WideResNet-28-10 — ~96%.
- **Требования к воспроизводимости:** Фиксированный seed (42), версия PyTorch 2.0+, отчёт с кодом и конфигурацией.
- **Правила сравнения:** Сравнение проводится по среднему значению accuracy за 3 запуска с разными seed. Разница менее 0.5% считается незначимой.
