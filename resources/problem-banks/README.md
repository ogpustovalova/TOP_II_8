# Банки задач и кейсов

Идеи практических, расчётных и кейсовых заданий для расширения КИМ и рубежного
контроля. При проектировании задач следуйте принципу «содержательное vs слабое
задание» (конспект ITMO Top AI, 2026-07-15, Блок 2): предпочитайте задания, где
студент должен показать эффект, сравнить варианты, обосновать выбор.

## Матрица «слабое vs содержательное задание»

| Объект проверки | Слабое задание | Содержательное задание |
|---|---|---|
| Регуляризация | Запустить модель с L2 | Показать переобучение и сравнить траектории качества при разных коэффициентах |
| Признаки | Обучить классификатор | Выявить утечку, построить признаки и обосновать их отбор |
| Метрики | Посчитать accuracy | Выбрать метрику под цену ошибок и исследовать порог решения |
| Модели | Сравнить три алгоритма | Задать единый протокол, оценить ресурсы и проанализировать ошибки |
| Оптимизаторы | Запустить Adam | Сравнить SGD/Adam/AdamW с единым seed, показать разную сходимость |

## Идеи кейсовых заданий

| Тема | Кейс | Связанные КИМ |
|---|---|---|
| Влияние архитектуры | Воспроизвести эксперимент He et al. 2015: сравнить plain CNN 20 и 56 слоёв на CIFAR-10, показать деградацию, добавить residual-блоки | [М4](../../M4-cnn/kim-01-cnn.md) |
| Связь свёртки и Фурье | Реализовать свёртку с нуля и показать её через теорему о свёртке в частотной области | [М4](../../M4-cnn/kim-01-cnn.md) |
| Self-attention (обзорно) | Реализовать self-attention `softmax(QKᵀ/√d)V` и визуализировать веса внимания | расширение М5 |
| Снижение размерности | Сравнить PCA и AE на одном датасете, показать, что AE даёт нелинейную проекцию | [М6](../../M6-autoencoders/kim-01-autoencoders.md) |
| Ландшафт потерь | Визуализировать ландшафт функции потерь для двухслойной сети при разных оптимизаторах | [М2](../../M2-training/kim-01-backprop-training.md), [М7](../../M7-optimizers/kim-01-optimizers.md) |

## Требования к добавлению

Для каждой задачи укажите проверяемый результат, сложность, входные данные,
эталон или подход к проверке и варианты параметризации.
