## Требования к добавлению

Для каждой задачи укажите проверяемый результат, сложность, входные данные, эталон или подход к проверке и варианты параметризации.

**Пример описания задачи для внутреннего банка лабораторных работ (Модуль 1):**

- **Название задачи:** Реализация многослойного перцептрона (MLP) на PyTorch/TensorFlow для классификации рукописных цифр (MNIST).
- **Проверяемый результат:** Студент создаёт и обучает MLP с заданной архитектурой, достигает accuracy на тестовом наборе ≥ 0.95, визуализирует кривые обучения, исследует влияние гиперпараметров (скорость обучения, количество слоёв, регуляризация) на качество.
- **Сложность:** Средняя (обязательная часть – реализация и обучение, продвинутая – эксперименты и анализ).
- **Входные данные:** Датасет MNIST (встроенный в PyTorch/TensorFlow).
- **Эталон / подход к проверке:** Проверяется работоспособность кода, соответствие архитектуры заданию, достижение порогового качества, наличие графиков и интерпретации. Дополнительные эксперименты оцениваются отдельно.
- **Варианты параметризации:** Можно варьировать число слоёв (2–5), количество нейронов, тип функции активации (ReLU, tanh), использование Dropout, выбор оптимизатора (SGD, Adam).
