# Методические указания преподавателям по использованию ресурсов

Как использовать материалы курса [`dlpython_course`](https://github.com/sozykin/dlpython_course), каталог ресурсов репозитория
и внешние источники при подготовке и проведении КИМ по дисциплине «Основы
нейронных сетей».

## 1. Исходные материалы курса

Ноутбуки курса [`dlpython_course`](https://github.com/sozykin/dlpython_course) — основной
источник готовых примеров для наполнения ноутбуков-заданий. Соответствие модулям:

| Модуль | Ноутбуки [`dlpython_course`](https://github.com/sozykin/dlpython_course) |
|---|---|
| 1. Искусственные НС | `01_introduction/fashion_mnist_dense.ipynb`, `01_introduction/pytorch/fashion_mnist_dense.ipynb` |
| 2. Обучение НС | `01_introduction/fashion_mnist_prevent_overfitting.ipynb` (кривые, регуляризация); PyTorch-вариант (явный цикл обучения) |
| 3. Полносвязные НС | `01_introduction/california_housing.ipynb` (регрессия), `01_introduction/fashion_mnist_dense.ipynb` (классификация) |
| 4. Сверточные НС | `02_cnn/` (CIFAR-10, Fashion-MNIST, animals), `03_pretrained_networks/` (ResNet), `04_transfer_learning/` (scenes, animals) |
| 5. Последовательности | `05_text_processing/`, `06_rnn/`, `07_cnn1d/`, `08_timeseries/weather_tensorflow.ipynb` |
| 6. Автокодировщики | готовых ноутбуков нет — собирается на основе теории (MNIST/Fashion-MNIST) |
| 7. Алгоритмы обучения | `keras_callbacks/lr_scheduler.ipynb`, `keras_callbacks/early_stopping.ipynb` |

При подготовке ноутбуков-заданий ключевые части заменяются на плейсхолдер
`# < ENTER YOUR CODE HERE >` по образцу `examples/lab-regression-features.ipynb`.

## 2. Три режима использования ресурсов

При проектировании и проведении КИМ ресурсы применяются в трёх режимах
(по методологии ITMO Top AI, Духанов, 09.07.2026):

- **Ручное использование** — преподаватель отбирает материалы (статью, датасет,
  бейзлайн) и встраивает их в КИМ вручную: аннотирует, формулирует задание,
  готовит рубрику.
- **Автоматизированное** — к КИМ прилагается готовый код: бейзлайн, скрипт
  проверки, шаблон ноутбука. Это ускоряет проверку и повышает воспроизводимость
  (например, автотест на порог точности, как в
  `examples/lab-regression-features.ipynb`).
- **Промпты для ИИ** — сформулированные промпты к большим языковым моделям как
  учебный инструмент: объяснить концепцию, сгенерировать учебный датасет, найти
  ошибку в коде. Промпты собраны в [`resources/llm-prompts/`](../../resources/llm-prompts/README.md)
  и должны сопровождаться оговорками об ограничениях LLM.

## 3. Каталог ресурсов репозитория

Каталог [`resources/`](../../resources/README.md) сгруппирован по типам. Каждый
раздел — таблица ресурсов с карточками: название, аннотация, связь с КИМ, доступ,
лицензия, дата проверки. При обновлении КИМ синхронизируйте ссылки на ресурсы.

## 4. Рекомендации по обновлению

- **Обновляйте курс ежегодно.** Концепции глубокого обучения меняются быстро;
  актуализируйте архитектуры, датасеты и статьи.
- **Следите за воспроизводимостью:** версии библиотек, API фреймворков, URL
  датасетов могут меняться — проверяйте ссылки перед семестром.
- **Балансируйте сложность:** при проектировании вариантов держите в уме
  матрицу «слабое vs содержательное задание» (конспект ITMO Top AI,
  2026-07-15.tex, Блок 2): предпочитайте задания, где нужно показать
  переобучение, сравнить траектории, выбрать метрику под цену ошибок, а не
  механически запустить модель с готовыми параметрами.
- **Сверяйте коды КРМ** при любом изменении привязки КИМ к компетенциям и
  индикаторам — используйте `data/krm-v3.0.xlsx` как источник истины.
