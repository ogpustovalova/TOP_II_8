# Методические указания преподавателям по оцениванию

Как оценивать КИМ, проводить устную защиту, давать обратную связь и работать с
апелляциями по дисциплине «Основы нейронных сетей».

## 1. Шкала и пороги

Оценивание ведётся по **единой 100-балльной шкале БРС УрФУ** (см.
[`docs/urfu-program-facts.md`](../../docs/urfu-program-facts.md); шкала
единообразна во всех ФОС УНЦ «ИИ»):

| Уровень | Баллы | Традиционная оценка | Зачёт | Качественный уровень |
|---|---|---|---|---|
| **В** (Высокий) | 80–100 | Отлично | Зачтено | Достигнуты в полном объёме, без замечаний |
| **С** (Средний) | 60–79 | Хорошо | Зачтено | В целом достигнуты, есть замечания без обяз. устранения |
| **П** (Пороговый) | 40–59 | Удовлетворительно | Зачтено | Достигнуты не в полной мере, есть замечания |
| **Н** (Недостаточный) | < 40 | Неудовлетворительно | Не зачтено | Существенные ошибки / задание не выполнено |

Порог зачёта — **40 баллов** (уровень не ниже П). Каждый КИМ оценивается от 0
до 100 по критериям с явными весами. Для каждого критерия используется
коэффициент уровня: Н = 0; П = 0,50; С = 0,75; В = 1,00. Балл по критерию равен
его весу, умноженному на коэффициент; баллы критериев суммируются.

> ⚠️ Пороговые значения метрик в рубриках (например «точность ≥ 80 %») —
> **предложены автором ФОС**, не из РПД УрФУ. Их можно убрать, оставив только
> качественные критерии по шкале В/С/П/Н. Подлежит согласованию с разработчиком
> курса.

## 2. Итоговая оценка по дисциплине. Технологическая карта БРС

> ⚠️ **Технологическая карта ниже предложена автором ФОС** по образцу ФОС других
> дисциплин УНЦ «ИИ» (см. [`docs/urfu-program-facts.md`](../../docs/urfu-program-facts.md),
> раздел 5–6). В РПД УрФУ для ОНС веса не заданы — конкретные коэффициенты
> подлежат согласованию с разработчиком курса.

Итоговая оценка формируется по схеме, принятой в УрФУ: коэффициенты значимости
по видам занятий (в сумме 1.0), внутри каждого — текущая аттестация и
промежуточная. По образцу ФОС «Машинное обучение», «ЦИНИ» (лекции + лабораторные):

### Технологическая карта БРС (5 семестр)

| Вид занятий | Коэффициент значимости | Внутреннее распределение |
|---|:---:|---|
| **Лекции** (17 ч) | **0,50** | текущая аттестация 0,50 + промежуточная (зачёт) 0,50 |
| **Лабораторные занятия** (34 ч) | **0,50** | текущая аттестация 1,00: среднее 7 лабораторных 0,50 + проект 0,50 |

Каждое мероприятие оценивается от 0 до 100 баллов внутри своего блока, затем
взвешивается коэффициентами.

### Перечень контрольно-оценочных мероприятий (предложено)

| Раздел | Мероприятие | Обозначение | Балл | Вес в итоге |
|---|---|:---:|:---:|:---:|
| Лекции, текущая | Контрольная работа по теоретическому материалу | К | 0–100 | 0,25 |
| Лекции, промежуточная | Зачёт | З | 0–100 | 0,25 |
| Лабораторные, текущая | Среднее арифметическое 7 лабораторных | Л | 0–100 | 0,25 |
| Лабораторные, текущая | Проектная работа с защитой | ПР | 0–100 | 0,25 |

Итоговый балл вычисляется по формуле:

`И = 0,25 × К + 0,25 × З + 0,25 × Л + 0,25 × ПР`.

При расчёте компонентов сохраняется один знак после запятой; итоговый балл
округляется до целого по математическим правилам.

### Допуск и положительная оценка

- К зачёту допускается обучающийся, сдавший все 7 лабораторных и проект, при
  `Л ≥ 40` и `ПР ≥ 40`.
- Положительный результат по дисциплине требует одновременно `З ≥ 40` и
  `И ≥ 40`.
- Результат 40–59 баллов является положительным и не требует пересдачи.
- При `З < 40` пересдаётся зачёт; при `Л < 40` или `ПР < 40` дорабатывается
  соответствующий компонент. После пересдачи итог пересчитывается по той же
  формуле.

## 3. Проведение устной защиты

Устная защита — обязательная часть сдачи каждого КИМ и основной надёжный
контроль в эпоху генеративного ИИ. Формат:

- 5–10 минут на КИМ, 7–10 минут на проект, 30–45 минут на зачёт.
- Минимальный набор вопросов приведён в `.md`-описании каждого КИМ; разрешается
  добавлять уточняющие.
- Ключевые приёмы проверки понимания:
  - вопросы «почему» (почему 3 слоя, а не 1; почему ResNet, а не plain; почему
    Adam, а не SGD);
  - просьба изменить параметр и спрогнозировать эффект;
  - просьба объяснить отдельный фрагмент кода строка за строкой.

Неответ учитывается при выборе уровня критерия защиты в соответствующей
рубрике. Отказ от ответа более чем по двум вопросам означает Н по критерию
понимания и ограничивает результат КИМ 39 баллами.

## 4. Банк контрольных вопросов

Вопросы сгруппированы по модулям в `kim-exam.md` (банк теоретических вопросов
зачёта годится и для защит лабораторных). Преподаватель может расширять банк,
следуя структуре: формулировка → ожидаемый ответ → типичная ошибка.

## 5. Обратная связь

- Укажите балльный уровень БРС (В/С/П/Н), подтвержденные КИУ уровня Б и
  конкретные критерии, по которым выставлен результат.
- Дайте 1–2 содержательных замечания, что улучшить (а не только оценку).
- При результате ниже 40 баллов — конкретные шаги для пересдачи: какие части
  КИМ переделать, какие вопросы повторить.

## 6. Апелляции

- Апелляция подаётся в установленные сроки; рассматривается по существу
  защиты, а не по ожиданию оценки.
- При апелляции студент заново отвечает на вопросы по оспариваемому КИМ;
  оценка может быть повышена, понижена или оставлена без изменений.
- Решение фиксируется со ссылкой на критерии рубрики.

## 7. Работа с несамостоятельностью

- Признаки (код без понимания на защите, дословное совпадение с LLM-ответом,
  идентичное решение у группы) — основание для дополнительных вопросов и/или
  снижения оценки.
- Решающее значение имеет устная защита: способность объяснить решение
  перевешивает любые подозрения по коду.
- Фальсификация метрик или использование чужих данных без разрешения —
  основание для пересдачи работы.
