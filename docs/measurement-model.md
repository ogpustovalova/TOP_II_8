# Модель измерения результатов обучения

Модель измерения связывает содержание дисциплины с компетенциями КРМ и доказательствами их освоения. Для каждого элемента должна существовать проверяемая связь «результат обучения → КИМ → критерий → балл».

## Условные обозначения

- **Б** — базовый уровень: понимание и выполнение типовых действий по инструкции;
- **С** — средний уровень: самостоятельное применение и обоснованный выбор методов;
- **П** — продвинутый уровень: проектирование, исследование, адаптация и решение нетиповых задач;
- **текущий контроль** — относится к теме или элементу модуля;
- **рубежный контроль** — подтверждает освоение модуля или группы модулей;
- **промежуточная аттестация** — фиксирует освоение дисциплины в целом.

## Правила заполнения

1. **Не копируйте формулировку компетенции целиком в дескриптор.** Укажите ту часть, которая реально формируется и проверяется в дисциплине.
2. **Один индикатор может проверяться несколькими КИМ**, если каждый КИМ покрывает различный аспект.
3. **Один КИМ может проверять несколько индикаторов**, но критерии должны показывать, какой балл относится к каждому результату.
4. **Уровень должен соответствовать сложности задания.** Нельзя заявлять продвинутый уровень, если обучающийся только воспроизводит алгоритм по инструкции.
5. Все ссылки должны вести на конкретные Markdown-файлы с заданиями, рубриками и ресурсами.

## Мини-проверка строки

- [ ] результат сформулирован через наблюдаемое действие;
- [ ] указан код компетенции и индикатора из КРМ;
- [ ] уровень Б/С/П согласован с дескриптором;
- [ ] КИМ действительно позволяет получить доказательство освоения;
- [ ] критерии оценивания измеряют заявленный результат;
- [ ] ресурсы доступны и легально используются.
