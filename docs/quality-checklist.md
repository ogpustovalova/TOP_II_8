# Чек-лист готовности репозитория

**Дата проверки:** 2026-07-19

**Подробный отчет:** [validation-report.md](validation-report.md)

Обозначения: `[x]` - выполнено; `[ ] Частично` - выполнено не полностью;
`[ ] Не выполнено` - блокирует готовность к публикации.

## Содержание

- [x] **Название дисциплины и аннотация заполнены.** Название и назначение есть
  в корневом `README.md`, аннотация приведена в `docs/rpd.md:49-54`.
- [x] **РПД содержит цель, задачи, пререквизиты и проверяемые результаты.** Все
  разделы присутствуют в `docs/rpd.md`; распределение часов по темам остается
  авторским и требует согласования.
- [x] **Выбраны целевые роли КРМ.** Для АИИ указаны ML Engineer и Data Analyst,
  для ПИИ - ML Engineer, Data Engineer и MLOps. Первичные «Модели выпускника»
  не включены в репозиторий.
- [ ] **Частично. Все коды компетенций и индикаторов сверены с КРМ версии 3.0.**
  Используемые коды существуют в `data/krm-v3.0.xlsx`, но уровни в README
  модулей расходятся с основной моделью, а заявление о дословном переносе
  дескрипторов не учитывает внесенные исправления.
- [ ] **Частично. Модель измерения заполнена для всех модулей и форм
  аттестации.** Семь модулей, проект и зачет отражены, но для заявленных
  рубежных срезов и лекционной контрольной готовых КИМ нет.
- [ ] **Частично. Каждый обязательный результат обучения подтверждается хотя бы
  одним КИМ.** Формальные ссылки есть, но DL-1.12 уровня С и DL-3.1 уровня П не
  подтверждаются фактическими заданиями; еще несколько КИУ покрыты лишь
  частично.
- [ ] **Частично. Каждый КИМ содержит условия, формат сдачи, критерии и шкалу.**
  Требуемые разделы есть, но рубрики не задают веса критериев и способ получения
  конкретного балла от 0 до 100.
- [ ] **Не выполнено. Баллы в рубриках согласованы с итоговой таблицей.** Проект
  с заявленным весом не менее 25 % отсутствует в технологической карте, а
  диапазон 40-59 в экзаменационной рубрике одновременно означает зачет и
  пересдачу.
- [x] **Описаны правила допустимого использования LLM и внешних ресурсов.** Они
  приведены в методических указаниях обучающимся и в каждом основном КИМ.
  Критерий сравнения со «средним решением группы» следует заменить
  воспроизводимыми признаками.
- [x] **Добавлены методические указания обучающимся и преподавателям.** Есть
  отдельные инструкции для обучающихся, оценивания и использования ресурсов;
  в методичке оценивания необходимо развести шкалы КРМ и БРС.
- [ ] **Частично. Ресурсы имеют аннотации и рабочие ссылки.** Большинство
  карточек заполнено, но две Kaggle-ссылки возвращают HTTP 404, а у части
  датасетов не указана точная лицензия.
- [ ] **Не выполнено. Указаны авторы, роли и вклад участников.** В
  `team/README.md` осталась только шаблонная строка без ФИО, ролей, вклада и
  контактов.
- [ ] **Не выполнено. Выбрана и согласована лицензия.** CC BY 4.0 объявлена для
  материалов, но в корневом README названа предлагаемой; лицензия кода не
  выбрана, права на исходные материалы не подтверждены.

## Техническая проверка

- [ ] **Частично. Нет незаполненных маркеров `[ЗАПОЛНИТЬ]` в публикуемой
  версии.** Буквальных незаполненных маркеров нет, но эквивалентные шаблонные
  поля остаются в `team/README.md`; несколько документов требуют согласования.
- [x] **Относительные Markdown-ссылки работают.** Проверены 264 локальные цели
  и один якорь, отсутствующих целей не найдено.
- [x] **Нет временных файлов офисных приложений.** Файлы блокировки,
  автосохранения и редакторские временные файлы не обнаружены.
- [x] **Крупные файлы не превышают ограничения GitHub или вынесены во внешнее
  хранилище.** Максимальный отслеживаемый файл имеет размер около 1,36 МБ;
  локальный архив CIFAR-10 около 170,5 МБ игнорируется Git.

## Общий вердикт

Репозиторий не готов к публикации как согласованный ФОС. Приоритетные блокеры:
трассируемость КИУ до заданий, воспроизводимый расчет итогового балла, авторство
и лицензирование. Полное обоснование и порядок исправлений приведены в
[отчете о валидации](validation-report.md).
