# Варианты формата КИМ (контрольно-измерительных материалов)

Зафиксированы варианты формата лабораторных работ / КИМ для курса
«Основы нейронных сетей». **Выбранный для реализации — вариант 4 (Гибрид).**

## Вариант 1. Ноутбук-задание + решение

Как пример по регрессии (`examples/lab-regression-features.ipynb`):
- `ipynb` с теорией и ячейками `# < ENTER YOUR CODE HERE >` для студентов.
- Отдельный `ipynb` с полным решением (`*-solution.ipynb`).

**Плюсы:** понятная механика сдачи, легко автопроверять, есть эталон.
**Минусы:** два файла на КИМ, теория дублируется или живёт отдельно, нет
формализованной рубрики/критериев оценки.

## Вариант 2. Методичка в ноутбуке

Как пример по вариационному исчислению (`examples/lab-variational-calculus.ipynb`):
- Один `ipynb` содержит и теорию, и примеры, и задания. Студент работает прямо в нём.

**Плюсы:** самодостаточность, низкий порог входа, теория и практика рядом.
**Минусы:** тяжёлые ноутбуки, нет отдельной рубрики, сложно версионировать критерии.

## Вариант 3. Описание КИМ в MD + ссылка на ноутбук

- `kim-NN-*.md` — описание по стандарту ФОС (назначение, результаты, критерии, шкала).
- Ноутбук-задание лежит в `attachments/` и **не перерабатывается** (как есть из `dlpython_course`).

**Плюсы:** минимум работы, чёткое разделение «документ / артефакт».
**Минусы:** КИМ не адаптирован под конкретные результаты обучения курса, нет
контроля в эпоху LLM (вопросов «почему», ручных датасетов), нет решения-эталона.

## Вариант 4 (ВЫБРАН). Гибрид: MD-описание + переработанный ноутбук-задание

Для каждого КИМ:
1. `kim-NN-<название>.md` — полное описание по стандарту ФОС:
   - назначение, проверяемый результат обучения (индикатор КРМ + уровень);
   - условия выполнения, формат сдачи;
   - критерии и рубрика оценивания (со шкалой и порогами);
   - правила использования внешних ресурсов и генеративного ИИ;
   - ссылка на ноутбук-задание и на эталонное решение.
2. `attachments/kim-NN-<название>.ipynb` — переработанный ноутбук-задание:
   - формат «задание + пустые ячейки» в стиле примера по регрессии;
   - контрольные вопросы «почему» (для защиты в эпоху LLM);
   - 来源 — переработка существующих ноутбуков `dlpython_course`.
3. `attachments/kim-NN-<название>-solution.ipynb` — эталонное решение (по мере готовности).

**Плюсы:** соответствие методологии КРМ/ФОС (дескриптор → КИМ → рубрика),
адаптация заданий под конкретные результаты обучения, контроль в эпоху LLM,
наличие эталона для преподавателя.
**Минусы:** наибольший объём работы, нужно перерабатывать существующие ноутбуки.

## Соответствие методологии КРМ

Вариант 4 напрямую реализует каскад «КИУ → дескриптор → задание → вариант»
из презентации Кустова (13.07.2026): MD-описание фиксирует дескриптор и
критерии, ноутбук реализует вариант задания, рубрика задаёт шкалу ассесмента.
