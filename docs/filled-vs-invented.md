## История исправленных допущений

### 1. Целевые роли

Я указал «Data Analyst и TADS». **По моделям выпускника УрФУ это ошибочно:**

| Программа | Роли (из «Модели выпускника») |
|---|---|
| АИИ (09.03.01) | **ML Engineer + Data Analyst** |
| ПИИ (09.03.03) | **ML Engineer + Data Engineer + MLOps** |

**TADS не фигурирует ни в одной программе.** Общая роль для обеих — ML Engineer.
ОНС обслуживает обе программы. В текущих документах указан фактический набор:
**ML Engineer, Data Analyst, Data Engineer, MLOps** с разделением по программам.

### 2. Уровни профиля выпускника ошибочно приписывались дисциплине

«Модель выпускника» задает итоговый ориентир всей образовательной программы,
но не доказывает достижение этого уровня одним курсом. Предыдущая версия
перенесла уровни С/П непосредственно в КИМ, хотя обязательные задания их не
измеряли.

| Индикатор | Целевой уровень профиля | Уровень текущего ФОС |
|---|:---:|:---:|
| DL-1.1 | С | Б |
| DL-1.2 | Б | Б |
| DL-1.3 | С | Б, проверяется в М4 |
| DL-1.4 | С | Б |
| DL-1.5 | П | Б |
| DL-1.7 | П | Б |
| DL-1.12 | С | Б |
| DL-3.1 | П | Б |
| DL-4.1 | С | Б |

Модель измерения теперь использует уровень Б и авторские предметные
дескрипторы наблюдаемых действий. Целевые уровни профиля сохранены как ориентир
для других дисциплин, практик и будущего расширения ФОС.

### 3. Шкала оценивания

В `methodical-guidelines/teachers-assessment/README.md` и всех рубриках я указал
пороги по Кустову (60/40) и уровни Б/С/П. **УрФУ использует свою шкалу:**

| Уровень | Баллы | Оценка | Зачёт |
|---|---|---|---|
| **В** (Высокий) | 80–100 | Отлично | Зачтено |
| **С** (Средний) | 60–79 | Хорошо | Зачтено |
| **П** (Пороговый) | 40–59 | Удовлетворительно | Зачтено |
| **Н** (Недостаточный) | < 40 | Неудовлетворительно | Не зачтено |

**Минимальный порог зачёта = 40 баллов** (не 60). Рубрики и методические
указания приведены к шкале УрФУ (В/С/П/Н, пороги 80/60/40).

## 🟡 Придумано мной, обосновано, но требует согласования

### 4. Часы по 7 темам
В `docs/rpd.md` графа часов **оставлена без разбивки по темам** — РПД её не
содержит. В v2 найден только общий объём (108 ч = 17 лекц. + 34 лаб. + 57 СРС).
**Распределение 17 лекционных и 34 лабораторных часов по 7 темам — моя
ответственность**, требующая согласования с автором курса. Можно предложить
распределение и зафиксировать его.

### 5. Веса в технологической карте БРС
В текущей технологической карте четыре компонента имеют вес по 25 %: среднее
семи лабораторных, проект, лекционная контрольная и зачёт. Схема сохраняет
коэффициенты лекций и лабораторных по 0,50 и внутреннее деление каждого блока
пополам. Лекционная контрольная реализована двумя срезами К1/К2 по 12,5 %
итоговой оценки каждый. Конкретные веса отсутствуют в исходной РПД и требуют
согласования.

### 6. Пороговые точности в рубриках (≥ 80/84/88 % на Fashion-MNIST и т. п.)
Эти пороги **остаются моими догадками** — в источниках их нет. Можно
сформулировать через шкалу В/С/П/Н без жёстких порогов точности, указывая
качественные критерии.

### 7. Длительности КИМ
«4–6 / 6–8 / 8–10 астрономических часов» — **мои оценки**, не из РПД. Можно
вывести из 34 лабораторных часов / 7 КИМ ≈ 5 ч на КИМ.

### 8. Контрольные вопросы и ситуационные задачи
Все вопросы «почему» в `kim-01-*.md` и банк вопросов экзамена — **написаны
мной** на основе тем. Один пример ситуационной задачи («99 % точность, но
случайное угадывание») — из лекции Гладилина (ITMO); остальные три — мои.
Методически корректны, но это не официальный банк УрФУ.

### 9. Архитектуры AE `784→128→64→32` — мой выбор
Типовая архитектура; в [`dlpython_course`](https://github.com/sozykin/dlpython_course) готового AE-ноутбука нет. Эксперимент
ResNet-20 vs ResNet-56 отсутствует в [`dlpython_course`](https://github.com/sozykin/dlpython_course); предложен как опциональное
расширение, что **честно отмечено** в `M4-cnn/kim-01-cnn.md`.

### 10. Промпты LLM в `resources/llm-prompts/`
Все 7 учебных промптов **написаны мной**. В изученных ФОС других дисциплин УрФУ
**раздел про использование ИИ отсутствует вообще** — это поле для собственной
разработки; мой раздел — опережающий вклад.
