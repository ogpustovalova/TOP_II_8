## 2. Цель и задачи

**Цель.** Сформировать у обучающихся практические навыки применения и разработки
архитектур глубоких нейронных сетей для решения прикладных задач — от математических
основ обучения до специализированных архитектур для изображений, последовательностей
и генеративных моделей.

**Задачи:**

1. Освоить математический аппарат нейронных сетей: модель нейрона, функции активации,
   расчёт градиентов, методы оптимизации, алгоритм обратного распространения ошибки.
2. Научиться обучать нейронные сети: функции потерь, градиентный спуск, борьба с
   переобучением, регуляризация.
3. Освоить полносвязные нейронные сети для задач регрессии и классификации.
4. Освоить сверточные нейронные сети для анализа изображений, включая современные
   архитектуры и перенос обучения.
5. Освоить нейронные сети для анализа последовательностей: рекуррентные сети и
   одномерные свёрточные сети для текстов и временных рядов.
6. Освоить автокодировщики (AE, VAE) для снижения размерности, обнаружения аномалий
   и генерации данных.
7. Освоить алгоритмы обучения нейронных сетей: современные оптимизаторы и управление
   скоростью обучения.

**Аннотация.** Практико-ориентированная дисциплина знакомит с моделью искусственного
нейрона и искусственными нейронными сетями, алгоритмами обучения глубоких нейронных
сетей, а также с простыми архитектурами: полносвязные, сверточные, рекуррентные,
автокодировщики. При реализации применяются технологии проблемного обучения,
проектный метод, кейс-метод, информационно-коммуникационные технологии, групповая
работа, исследовательские методы. Основные фреймворки — PyTorch и TensorFlow.

## 5. Система оценки результатов обучения

Система оценки построена по методологии КРМ v3.0 и реализована в виде
интерактивной таблицы **модели измерения** в [главном `README.md`](../README.md)
(раздел 2). Состав оценивающих средств:

- **Текущий контроль:** лабораторные работы (КИМ) по каждому из 7 модулей —
  ноутбуки с защитой.
- **Рубежный контроль:** лекционная контрольная и проектная работа с защитой.
- **Промежуточная аттестация:** зачёт (5 семестр).

Итоговая оценка вычисляется по формуле
`И = 0,25 × Л + 0,25 × ПР + 0,25 × К + 0,25 × З`, где Л — среднее по семи
лабораторным, ПР — проект, К — лекционная контрольная, З — зачёт. Все компоненты
оцениваются по шкале 0–100. Для допуска нужны `Л ≥ 40` и `ПР ≥ 40`; для зачёта —
`З ≥ 40` и `И ≥ 40`.

Оценивание ведётся по **единой 100-балльной шкале БРС УрФУ** (см.
[`docs/urfu-program-facts.md`](urfu-program-facts.md) и образцы ФОС других
дисциплин УНЦ «ИИ»):

| Уровень | Баллы | Традиционная оценка | Зачёт |
|---|---|---|---|
| **В** (Высокий) | 80–100 | Отлично | Зачтено |
| **С** (Средний) | 60–79 | Хорошо | Зачтено |
| **П** (Пороговый) | 40–59 | Удовлетворительно | Зачтено |
| **Н** (Недостаточный) | < 40 | Неудовлетворительно | Не зачтено |

Порог зачёта — 40 баллов; диапазон 40–59 является положительным. Подробные
критерии и рубрики приведены в КИМ
соответствующих модулей и зачёта; технологическая карта БРС для дисциплины — в
[методических указаниях преподавателям](../methodical-guidelines/teachers-assessment/README.md).
Правила использования генеративного ИИ и внешних ресурсов сформулированы в
[методических указаниях обучающимся](../methodical-guidelines/students/README.md).
