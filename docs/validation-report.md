## 3. Основание и методика проверки

Учебник подключает проверенные главы через обертки в `textbook/chapters/`.
Для валидации использованы прежде всего главы, посвященные КРМ, РПД, ФОС,
оцениванию, LLM и открытому репозиторию:

- `textbook/chapters/2026-07-09.tex`: обязательные части РПД, модель измерения,
  состав ФОС, КИМ, ресурсы и методические указания;
- `textbook/chapters/2026-07-13.tex`: каскад
  `роль -> компетенция -> индикатор -> уровень -> дескриптор -> задание`;
- `textbook/chapters/2026-07-15.tex`: согласованность результата, роли, веса
  проекта, воспроизводимость и итоговый чек-лист экспертизы;
- `textbook/chapters/2026-07-16.tex`: оценивание полного цикла работы, правила
  использования GenAI, рубрики и воспроизводимость;
- `textbook/chapters/2026-07-17.tex`: README, лицензия, тесты, сопровождение и
  пригодность репозитория для внешнего пользователя.

Главный примененный принцип: каждый заявленный результат должен быть
трассируем до наблюдаемого действия в КИМ и обратно агрегироваться через
рубрику до конкретного КИУ. Правдоподобные, но не подтвержденные первичным
источником сведения не считались установленными фактами.

Проверены все Markdown-файлы, 14 ноутбуков, 16 Python-скриптов,
книга `data/krm-v3.0.xlsx`, исходная РПД `other/РПД_ОНС.docx`, локальные ссылки,
структура Git и размеры файлов. Все ноутбуки проверены на валидность формата и
синтаксис code-ячеек автоматическим валидатором `tools/validate_repository.py`.
Семь эталонов исполнены целиком в чистом окружении; семь студенческих
ноутбуков не исполнялись, поскольку содержат обязательные плейсхолдеры.

## 5. Трассируемость КРМ и КИМ

Коды индикаторов существуют в приложенной КРМ. Целевые уровни профиля
выпускника С/П отделены от фактически измеряемого дисциплинарным ФОС уровня Б.
Предметный дескриптор в КИМ фиксирует наблюдаемые действия КИУ. Полная матрица
`дескриптор -> обязательное действие -> артефакт -> пороговый критерий` приведена
в [`kiu-coverage.md`](kiu-coverage.md).

| DL-1.2, Б | В модулях 1 и 3 студент реализует и обучает MLP. | Покрыт |

Результаты корректировки:

- Уровни Б синхронизированы в модели измерения, РПД, README модулей, КИМ,
  генераторах и студенческих ноутбуках.
- В M2 обязательны выбор активации, инициализации и связности; в M4 — few-shot,
  YOLO/OpenCV-видео и save/load; в M5 — LSTM/GRU и полный классический NLP.
- Лекционная контрольная реализована срезами К1/К2 с вариантами, ключами и
  общей рубрикой; она проверяет понимание, не заменяя лабораторные свидетельства.
- Проект подтверждает только КИУ, заранее выбранные в согласованном плане, а не
  автоматически все индикаторы из общего перечня.
- Зачет агрегирует ранее полученные практические свидетельства; устные вопросы
  больше не объявлены самостоятельным подтверждением всех КИУ.

## 6. Оценивание и воспроизводимость

Все десять рубрик содержат веса критериев с суммой 100. Балл критерия равен его
весу, умноженному на коэффициент уровня: Н = 0; П = 0,50; С = 0,75; В = 1,00.
Уровень Н по ключевому критерию понимания ограничивает результат КИМ 39
баллами.

Технологическая карта задает четыре равновесных компонента: среднее семи
лабораторных, проект, лекционная контрольная и зачет. Формула итоговой оценки:
`И = 0,25 × Л + 0,25 × ПР + 0,25 × К + 0,25 × З`. Проектная защита входит
только в проектную рубрику. Для контрольной `К = (К1 + К2) / 2`; оба среза
необходимо выполнить не ниже 40 баллов. Для положительного результата необходимы
`З ≥ 40` и `И ≥ 40`; диапазон 40–59 не требует пересдачи.

## 7. Ресурсы и технические замечания

- Неработающие ссылки на `ml-thread-4-2026`, Intel Image Classification в Kaggle
  и фактически недоступную главную страницу eLibrary заменены рабочими
  источниками. Ранее отдельно проверены 37 ссылок ресурсного контура; финальный
  валидатор проверил 50 уникальных URL всех Markdown-файлов. Доступны 48;
  `asozykin.ru` вернул HTTP 403, а `yann.lecun.com` — ошибку TLS. Оба результата
  записаны как сетевые предупреждения, подтверждённых HTTP 404/410 нет.
- Карточки больше не приписывают CIFAR-10 лицензию MIT и не переносят лицензии
  Keras/torchvision на данные. Для наборов без отдельной лицензии установлен
  запрет на включение копий в репозиторий; Intel, Animals и исходный `banks.csv`
  исключены из обязательных КИМ.
- Для `banks.csv` подтверждены действующая загрузка из исходного ноутбука,
  структура 13 999 записей, SHA-256 и лицензия MIT.
- M5 использует закреплённый revision Russian Reviews Classification; загрузка
  `train.jsonl` выполняется через временный файл с проверкой SHA-256. Эталон по
  умолчанию работает с фиксированной подвыборкой опубликованного train, а
  синтетика оставлена только как явно помеченный автономный режим. Метаданные
  набора указывают Apache-2.0, но не раскрывают происхождение текстов.
- `repository-tree.txt` обновлён и отражает актуальную структуру репозитория.
- `docs/cache-documentation.md` содержит 10 карточек датасетов и кэшей с локальным
  путём, источником, лицензией, revision, SHA-256, командами загрузки, проверки
  и очистки.
- Игнорирование данных ограничено каталогами `M*/attachments/data/`; новые
  первичные файлы в корневом `data/` больше не скрываются Git.
- В рабочей копии находится около 824 МБ игнорируемых датасетов и кэшей. В Git
  они не попадают. Процедура загрузки, проверки и очистки задокументирована
  в `docs/cache-documentation.md`.
