# Рубрика «Проектная работа»

Шкала: **Б** / **С** / **П**. Итоговый уровень = минимум из «Корректность» и
«Анализ ошибок». Перевод в баллы: **Б → 60–74**, **С → 75–89**, **П → 90–100**.
Порог зачёта проекта — не ниже Б. Вес проекта в итоговой оценке — не менее 25 %.

| Критерий | Б (60–74) | С (75–89) | П (90–100) |
|---|---|---|---|
| **Корректность и постановка** | Задача поставлена, модель обучена, получены измеримые результаты | То же + постановка обоснована, выбрана адекватная архитектура и метрика, нет утечек в валидации | То же + сравнение нескольких архитектур/стратегий, статистическая устойчивость результатов |
| **Воспроизводимость** | Код запускается, есть `README` и базовая инструкция | То же + `requirements.txt`, зафиксированный seed, чистая структура репозитория | То же + CI/скрипты запуска одного эксперимента, кэширование данных, портативность |
| **Анализ ошибок** | Приведена итоговая метрика | То же + confusion matrix / разбор типовых ошибок, обсуждение слабых мест | То же + содержательный анализ причин ошибок, предложения по улучшению, оценка цены ошибок |
| **Оригинальность** | Решение в основном следует шаблону / типовому подходу | То же + нестандартные ходы в части архитектуры, признаков или пайплайна | То же + решение заметно отличается от типового LLM-ответа и среднего решения группы; есть собственная находка |
| **Качество защиты** | Демонстрирует работающую модель, отвечает на базовые вопросы | То же + обосновывает выбор архитектуры и метрики, реагирует на уточняющие вопросы | То же + глубоко владеет материалом, отвечает на вопросы «почему», корректно обсуждает ограничения |

## Контроль устной защиты

Обязательна (7–10 минут + вопросы). Минимальный набор вопросов — в
[`kim-project.md`](kim-project.md). Неответ на ключевой вопрос (например, о
валидации или метрике) снижает итоговый уровень на ступень.

## Уровни по КРМ и балльная нормировка

| Уровень | Баллы | Статус ассесмента |
|---|---|---|
| П (продвинутый) | 90–100 | достигнут |
| С (средний) | 75–89 | достигнут |
| Б (базовый) | 60–74 | достигнут (минимальный порог курса) |
| Ниже Б | < 60 | не достигнут — доработать и пересдать |
